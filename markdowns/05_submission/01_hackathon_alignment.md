# Hackathon Alignment — AMD Developer Hackathon ACT II Unicorn Track

## Track understanding

The Unicorn Track is product/startup-oriented. It is not a leaderboard benchmark. Judging focuses on:

- Creativity and originality
- Product/market potential
- Completeness
- Meaningful use of AMD platforms

Submissions also need to be containerized, have a public GitHub repository, and include setup/usage instructions.

## Why A+Prompt fits

### Creativity and originality

A+Prompt is not another chatbot wrapper or prompt library. It introduces an interactive AI gym metaphor:

- Prompt reps
- Form correction
- Difficulty progression
- Score-based coaching
- Before/after output comparison
- Agent unlock

The novel behavior is that users learn AI by solving their own business tasks, not by watching passive lessons.

### Product/market potential

The market is large because AI adoption is spreading quickly, but effective usage is uneven. Small-business owners need practical AI workflows for customer service, marketing, research, and operations.

A+Prompt can become:

- B2C subscription for business owners.
- B2B training platform for SME communities.
- White-label AI learning tool for bootcamps/consultants.
- Vertical AI workflow marketplace.

### Completeness

MVP demonstrates full product loop:

```text
business mission → user prompt → score → coaching → improved prompt → before/after output → saved playbook → supervised agent
```

This is a complete learning and productivity loop, not only a prototype screen.

### Use of AMD platforms

A+Prompt can use AMD platforms in two levels:

1. **Fireworks AI API credits** for LLM inference, powered through AMD/Fireworks infrastructure.
2. **Optional AMD GPU deployment** for open-source model serving through ROCm-compatible stack.

The AI calls are meaningful:

- Prompt Judge
- Prompt Coach
- Rewrite Coach
- Output Simulator
- Customer Save Agent

## Recommended submission wording

> A+Prompt uses AMD-backed Fireworks inference to power a multi-agent AI training loop for business users. Instead of optimizing a benchmark, we use open-source models to run practical AI coaches: a Prompt Judge, Rewrite Coach, Output Simulator, and Customer Save Agent. This demonstrates a startup-oriented application of AMD AI infrastructure for real-world AI upskilling.

## What to show judges

1. Show weak prompt.
2. Show low score.
3. Show coach correction.
4. Show improved prompt.
5. Show better output.
6. Show saved playbook.
7. Show Customer Save Agent.
8. Explain AMD/Fireworks model layer.

## What not to overclaim

Do not claim:

- Fully autonomous customer support.
- Guaranteed business revenue increase.
- Fine-tuned model if not actually fine-tuned.
- Official Karpathy or Andrew Ng curriculum.

Say instead:

- Inspired by AI power-user principles.
- Source-backed curriculum.
- Supervised agent workflow.
- Designed for future fine-tuning once user feedback data is collected.
